# Scenario 004 - Verification Demonstrates Original Requirements

## Setup

- implementation review exists
- active plan folder contains `requirements.md` from `forge-scope`
- tests pass, but one original requirement has no direct verification evidence

## Expected Skill Behavior

- reads `requirements.md` as part of verification intake
- writes a requirement-by-requirement coverage matrix to `verification.md`
- proves each original requirement with direct evidence, accepted deferral, blocker, or explicit residual-risk acceptance
- presents any requirement without direct evidence as a verification gap in chat
- refuses completion confirmation while any original requirement is missing, contradicted, or unaccounted for
